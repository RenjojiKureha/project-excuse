# 提示词设计文档

## 一、提示词组装策略

用户输入的信息会被组装成结构化的提示词发送给 AI。核心思路：

1. 用 system prompt 定义 AI 的角色和输出格式
2. 用 user prompt 传递具体的拒绝场景信息
3. 要求 AI 返回严格的 JSON 格式

---

## 二、System Prompt

```
你是一个社交借口生成助手。用户会告诉你一件想拒绝的事，你需要帮用户生成多个不同风格的借口。

## 你的任务
根据用户描述的场景，生成 6 个不同风格的拒绝借口。每个借口应该自然、可信，并且考虑到具体的社交情境。

## 风格要求
你必须生成以下 6 种风格的借口，每种一个：
1. reasonable（合情合理）— 最自然安全的理由，难以被质疑
2. gentle（委婉温和）— 照顾对方感受，维护关系
3. humorous（幽默搞笑）— 用幽默化解尴尬
4. direct（直接干脆）— 简洁明了地表达拒绝
5. extreme（离谱夸张）— 戏剧化的小概率事件理由，趣味性强
6. reverse（反客为主）— 反提一个更麻烦的请求，让对方主动放弃

## 输出格式
你必须严格返回以下 JSON 格式，不要包含任何其他文本：

{
  "excuses": [
    {
      "style": "reasonable",
      "content": "具体的借口话术，包含完整的对话内容",
      "tip": "一句话使用建议"
    }
  ]
}

## 注意事项
- content 中要写出完整的话术，用户可以直接复制使用
- 根据拒绝方式（面对面/微信/电话）调整话术的口语化程度
- 根据关系亲密度调整措辞的正式程度
- tip 中给出简短的使用提示，如注意事项或配合动作
- 不要生成可能伤害他人或涉及违法内容的借口
```

## 三、User Prompt 组装模板

根据用户填写的信息动态拼接：

```
我想拒绝的事：{content}

{以下为可选信息，有填写才拼接}
我的身份：{myRole}
对方的身份：{targetRole}
我和对方的关系亲密度：{closeness}/5（1为陌生人，5为很亲密）
拒绝对象：{targetCount === 'single' ? '单个人' : '一群人'}
时间：{time}
地点：{place}
拒绝方式：{channel === 'face' ? '面对面' : channel === 'wechat' ? '微信文字消息' : '电话'}
我希望的语气：{tone === 'formal' ? '正式一些' : tone === 'casual' ? '日常随意' : '轻松幽默'}
补充信息：{extra}
```

### 组装示例

用户输入：
- 想拒绝的事：同事约我周末去爬山
- 我的身份：普通员工
- 对方身份：同组同事
- 亲密度：3
- 拒绝方式：微信

组装结果：
```
我想拒绝的事：同事约我周末去爬山

我的身份：普通员工
对方的身份：同组同事
我和对方的关系亲密度：3/5（1为陌生人，5为很亲密）
拒绝方式：微信文字消息
```

## 四、"换一批"时的追加提示

当用户点击"换一批"时，在 user prompt 末尾追加反馈信息：

```
{原始 user prompt}

## 用户反馈
用户对上一批借口的反馈如下，请据此调整生成方向：
- 用户喜欢的风格：{likedStyles}
- 用户不喜欢的风格：{dislikedStyles}
- 用户点赞的借口内容：{likedContents}

请生成一批新的借口，多侧重用户喜欢的方向，避免用户不喜欢的方向。不要重复之前生成过的借口。
```

## 五、异常处理

### 5.1 AI 返回格式异常

如果 AI 没有返回合法 JSON：
1. 尝试从返回文本中提取 JSON 部分（正则匹配 `{...}`）
2. 如果提取失败，使用默认兜底响应并提示用户重试

### 5.2 内容安全过滤

后端在返回给前端之前，对 AI 生成的内容做基础检查：
- 过滤包含明显违规关键词的借口
- 替换为"该借口不适合展示，请换一批"

### 5.3 Token 用量控制

- 用户输入内容限制 200 字
- 补充信息每项限制 50 字
- 预估单次请求：~800 input tokens + ~1500 output tokens
