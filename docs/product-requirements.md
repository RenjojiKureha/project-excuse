# 产品需求文档

## 一、用户输入设计

### 1.1 主输入（必填）

- **想拒绝的事**：用户用自然语言描述想拒绝的事情
  - 输入框支持自由文本，placeholder 示例："同事约我周末去爬山"
  - 最大 200 字

### 1.2 补充输入（全部可选，折叠面板展示）

| 字段 | 说明 | 输入方式 | 示例 |
|------|------|----------|------|
| 我的身份 | 用户自身角色 | 文本输入 | 普通员工、学生、儿媳 |
| 对方身份 | 被拒绝者的角色 | 文本输入 | 直属领导、同学、婆婆 |
| 关系亲密度 | 和对方的亲密程度 | 滑块（1-5） | 1=陌生人，5=亲密关系 |
| 拒绝对象人数 | 拒绝单人还是群体 | 单选 | 单人 / 多人 |
| 时间 | 事件发生的时间 | 文本输入 | 今晚、下周六、春节 |
| 地点 | 事件发生的地点 | 文本输入 | 公司、老家 |
| 拒绝方式 | 通过什么渠道拒绝 | 单选 | 面对面 / 微信文字 / 电话 |
| 语气偏好 | 想要什么样的语气 | 单选 | 正式 / 日常 / 轻松 |
| 自定义补充 | 用户额外补充的信息 | 文本输入 | "上次已经拒绝过一次了" |

### 1.3 预设场景快捷入口

首页提供常见拒绝场景的快捷卡片，点击自动填充主输入和部分补充信息：

- 拒绝聚餐/饭局
- 拒绝加班
- 拒绝借钱
- 拒绝帮忙
- 拒绝约会/相亲
- 拒绝送礼/收礼
- 拒绝出席活动
- 拒绝推销/安利

用户也可以直接跳过快捷入口，自由输入。

## 二、借口风格分类

AI 生成时按以下风格分类，每种风格至少生成 1 个借口：

| 风格标签 | 说明 | 图标色 |
|----------|------|--------|
| 合情合理 | 最自然、最安全的借口，不容易被拆穿 | 绿色 |
| 委婉温和 | 照顾对方感受，不伤关系 | 蓝色 |
| 幽默搞笑 | 用幽默化解尴尬，适合关系好的人 | 黄色 |
| 直接干脆 | 不绕弯子，简洁表达拒绝 | 橙色 |
| 离谱夸张 | 小概率事件、戏剧化的理由，趣味性强 | 紫色 |
| 反客为主 | 反过来提出一个更麻烦的请求，让对方主动放弃 | 红色 |

## 三、结果展示页

### 3.1 借口卡片

每个借口以卡片形式展示，包含：

- **风格标签**（带颜色和图标）
- **借口正文**（AI 生成的具体话术）
- **使用建议**（简短的使用提示，如"适合微信回复"、"注意语气要自然"）
- **操作按钮**：
  - 👍 点赞 — 标记为好用
  - 👎 点踩 — 标记为不好用
  - 📋 复制 — 一键复制借口文本
  - ⭐ 收藏 — 保存到个人收藏
  - 🖼️ 分享图片 — 将该借口生成为图片并分享（详见 3.3）

### 3.2 全局操作

- **换一批**：根据点赞/点踩的偏好，重新生成一批借口
  - 换一批时，向 AI 传递用户的反馈偏好（喜欢哪种风格、不喜欢哪种）
- **微调输入**：不离开结果页，直接修改补充信息后重新生成

### 3.3 分享图片功能

用户可以将单条借口生成为精美的分享图片，长按或点击即可保存/转发。

**图片内容布局（从上到下）：**

```
┌──────────────────────────┐
│      ExcuseMe 借口生成器   │  ← 顶部 App 品牌标识
│──────────────────────────│
│  「合情合理」               │  ← 风格标签（带颜色）
│                          │
│  不好意思，这周末我答应陪   │
│  家里人去医院做体检，已经   │  ← 借口正文
│  约好了不太好改时间...      │
│                          │
│  💡 语气自然，不用解释太多  │  ← 使用建议
│──────────────────────────│
│                   ┌─────┐│
│  扫码生成你的借口   │ QR  ││  ← 底部：引导文案 + 二维码
│                   │code ││
│                   └─────┘│
└──────────────────────────┘
```

**设计要点：**
- 图片宽度固定 375px（移动端屏幕宽），高度自适应内容
- 背景使用品牌色渐变或纯色，根据借口风格变化配色
- 二维码指向本站首页（或带参数的推广链接），v1 先预留二维码位置，使用占位图
- 二维码区域设计为独立模块，后续可替换为动态生成的二维码
- 生成图片后弹出预览浮层，提供"保存到相册"和"分享"操作
- 移动端使用浏览器原生分享 API（`navigator.share`），不支持时降级为"长按保存图片"提示

## 四、历史记录

- 自动保存每次生成的记录（输入内容 + 生成结果）

- 支持按风格标签筛选

## 六、用户体验细节

### 6.1 输入阶段

- 主输入框自动聚焦，引导用户直接输入
- 补充信息折叠展示，标题显示"补充更多信息，生成更精准的借口"
- 输入过程中实时显示字数统计
- 支持键盘 Enter 直接提交

### 6.2 生成阶段

- 显示生成动画 + 趣味等待文案（如"正在绞尽脑汁帮你想借口..."）
- 流式输出，借口卡片逐个出现

### 6.3 结果阶段

- 复制成功后显示短暂提示
- 点赞/点踩有动画反馈
- 换一批有平滑的卡片过渡动画

### 6.4 移动端 Web 设计

本项目为**移动端 Web 应用**，设计和开发均以移动端为唯一目标：

- 视口宽度按 375px 设计，使用 rem/vw 做适配
- 所有交互按触屏操作设计（点击区域不小于 44px、无 hover 态依赖）
- 底部导航栏固定，包含：首页 / 历史 / 收藏
- 页面切换使用左右滑动过渡动画
- 输入框使用移动端友好的键盘弹起适配
- 图片分享预览浮层全屏覆盖，支持手势关闭
